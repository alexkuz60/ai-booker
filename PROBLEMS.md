# Проблемы и противоречия (рабочий файл)

> Временный файл для отслеживания обсуждения. Решённые пункты заменяются финальными формулировками.

## А. Синхронизация данных — ✅ РЕШЕНО

**Финальная формулировка (записана в ARCHITECTURE.md §1):**

- Исходный PDF **никогда** не покидает устройство пользователя. Хранится как `source/book.pdf` в ProjectStorage.
- На сервер уходят **только**: извлечённые текстовые блоки (для ИИ-анализа и TTS) и структурные метаданные (при ручном пуше).
- Жизненный цикл: **Инициализация** → **Local-auto-save** (мгновенно, без обращения к БД) → **Manual-push** (кнопка «На сервер»).
- Термин «dual-write» удалён. Начальная запись при upload — это единоразовая инициализация, не постоянный паттерн.

## Б. Разрыв данных персонажей: Парсер ↔ Студия

**Проблема:** Парсер хранит персонажей локально (`structure/characters.json` — `LocalCharacter[]`), а Студия работает с серверной таблицей `book_characters`. Правки в Парсере (алиасы, пол, описание) не попадают в Студию автоматически.

**Что не определено:**
1. Когда именно локальные персонажи должны синхронизироваться в `book_characters`? При пуше «На сервер»? При переходе в Студию?
2. Обратная синхронизация: если Студия обогатила профиль (temperament, voice_config, speech_style) — должно ли это отражаться в локальном файле?
3. Формат `LocalCharacter` беднее, чем `book_characters` (нет temperament, speech_style, voice_config, color, sort_order). Как мерджить?

**Статус:** 🔴 Ожидает обсуждения

## В. Маршрутизация ИИ-провайдеров в Edge Functions — ✅ РЕШЕНО

**Финальная формулировка (записана в ARCHITECTURE.md §3):**

- Все edge functions должны поддерживать пользовательский роутинг (клиент передаёт `model` + `apiKey`).
- Lovable AI — высший приоритет **только для админов**.
- Для пользователей приоритет: директ-подписка → OpenRouter → DotPoint ≡ ProxyAPI.
- Фоллбек на сервере **не выполняется** — ошибка возвращается клиенту, клиент ищет аналог по пользовательским спискам.
- TTS-модели имеют отдельную логику (будет определена позже).
- **Текущий статус реализации:** `extract-characters` и `parse-book-structure` поддерживают роутинг; `segment-scene`, `profile-characters`, `detect-inline-narrations`, `generate-atmosphere-prompt` — требуют обновления.

## Е. Неопределённость формата передачи контекста Парсер → Студия

**Проблема:** Переход между модулями использует `sessionStorage` для передачи `bookId`, `chapterId`, активной вкладки. Формальный контракт отсутствует.

**Статус:** 🔵 Отложено (не критично на текущем этапе, нет сторонних посетителей)

**Важно — сохранить для анализа:**
- `localStorage` / `sessionStorage` ключи Студии (`studio_selected_scene_idx`, `studio_active_tab`, `studio_chapter`) — **не трогать**.
- Записи `user_settings` в БД (ключ `studio_session`) — **не трогать**.
- Эти данные пригодятся для анализа формата, который нужно будет воспроизвести в новом локальном хранилище.

По итогам Фаз 1–2 станет ясно: какие роли задействованы, какие edge functions им нужны, какие промпты использовать, где применяются гибридные (хардкодовые) алгоритмы вместо AI.

### План исправлений (порядок работы)

#### Волна 1 — Критичные (блокируют пользователей)

**B5: Merge уничтожает иерархию PDF outline**
- Корень: `mergeOutlineWithTextToc` вызывает `flatten(outline)` → теряет `children`
- Исправление: вставлять новые текстовые записи в существующую иерархию, не уплощая outline
- Файл: `src/lib/pdf-extract.ts`

**B1/B6: Reload → дубликат книги**
- Корень: `handleFileSelect` всегда делает INSERT, не учитывая текущий `bookId`
- Исправление: если `bookId` уже существует → UPDATE `books` + удалить старые chapters/parts/scenes, а не создавать новую запись
- Файл: `src/hooks/useBookManager.ts`

**B4/B7: DOCX-mode теряется после page refresh**
- Корень: нет персистентного признака «книга = DOCX» (ни в локале, ни в state)
- Исправление: сохранять `fileFormat: "pdf" | "docx"` в `project.json` и `structure/toc.json`; `useChapterAnalysis` проверяет формат перед вызовом `ensurePdfLoaded`
- Файлы: `useBookManager.ts`, `useChapterAnalysis.ts`, `localSync.ts`

#### Волна 2 — Средние (UX-проблемы)

**B2: Folder-ноды остаются pending**
- Исправление: в `handleFileSelect` после `sanitizeChapterResultsForStructure` дополнительно пройти по всем folder-нодам и выставить `done`
- Файл: `src/hooks/useBookManager.ts`

**B3: sessionStorage не очищается при reload**
- Исправление: `reloadBook` должен удалять `docx_chapter_texts`, `docx_html`
- Файл: `src/hooks/useBookManager.ts`

**B8: Молчаливый null при отсутствии PDF на сервере**
- Исправление: если `pdfBlob = null` и книга НЕ DOCX → показать toast-предупреждение
- Файл: `src/hooks/useBookManager.ts`
