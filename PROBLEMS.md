# Проблемы и противоречия (рабочий файл)

> Временный файл для отслеживания обсуждения. Решённые пункты заменяются финальными формулировками.

## А. Синхронизация данных — ✅ РЕШЕНО

**Финальная формулировка (записана в ARCHITECTURE.md §1):**

- Исходный PDF **никогда** не покидает устройство пользователя. Хранится как `source/book.pdf` в ProjectStorage.
- На сервер уходят **только**: извлечённые текстовые блоки (для ИИ-анализа и TTS) и структурные метаданные (при ручном пуше).
- Жизненный цикл: **Инициализация** → **Local-auto-save** (мгновенно, без обращения к БД) → **Manual-push** (кнопка «На сервер»).
- Термин «dual-write» удалён. Начальная запись при upload — это единоразовая инициализация, не постоянный паттерн.

## Б. Разрыв данных персонажей: Парсер ↔ Студия

**Проблема:** Парсер хранит персонажей локально (`structure/characters.json` — `LocalCharacter[]`), а Студия работает с серверной таблицей `book_characters`. Правки в Парсере (алиасы, пол, описание) не попадают в Студию автоматически.

**Что не определено:**
1. Когда именно локальные персонажи должны синхронизироваться в `book_characters`? При пуше «На сервер»? При переходе в Студию?
2. Обратная синхронизация: если Студия обогатила профиль (temperament, voice_config, speech_style) — должно ли это отражаться в локальном файле?
3. Формат `LocalCharacter` беднее, чем `book_characters` (нет temperament, speech_style, voice_config, color, sort_order). Как мерджить?

**Статус:** 🔴 Ожидает обсуждения

## В. Маршрутизация ИИ-провайдеров в Edge Functions

**Проблема:** `extract-characters` обновлён для роутинга через пользовательские ключи (OpenRouter / ProxyAPI). Но остальные аналитические функции работают **только** через Lovable Gateway (admin-ключ):
- `segment-scene`
- `profile-characters`
- `detect-inline-narrations`
- `generate-atmosphere-prompt`

**Что не определено:**
1. Должны ли все edge functions поддерживать пользовательский роутинг? Или часть остаётся admin-only?
2. Если пользователь задал свой OpenRouter-ключ, но вызывает `segment-scene` — что происходит? Молчаливый фоллбек на Lovable Gateway? Ошибка?

**Статус:** 🔴 Ожидает обсуждения

## Г. Inline-нарротации: отсутствие пользовательского контроля

**Проблема:** `detect-inline-narrations` определяет интонационные пометки внутри фраз, но результат:
- Не показывается пользователю для ревью/правки
- Не имеет выделенной AI-роли в `aiRoles.ts`
- Непонятно, сохраняется ли в `segment_phrases.metadata` или теряется

**Нарушает принцип:** пользователь контролирует все промежуточные результаты.

**Что не определено:**
1. UI для просмотра/правки inline-нарротаций (в Раскадровке?)
2. Какая AI-роль отвечает за детект? «Режиссёр»? Отдельная роль?
3. Формат хранения в metadata и отображение в `PhraseRenderer`

**Статус:** 🔴 Ожидает обсуждения

## Д. Терминология WORKFLOW.md: «Навигатор автосинхронизирует с БД»

**Проблема:** WORKFLOW.md утверждает, что навигатор (NavSidebar) автоматически синхронизирует изменения с БД. Это **неверно** — NavSidebar работает исключительно с React-стейтом и OPFS. Запись в БД происходит только по кнопке «На сервер».

**Статус:** 🟡 Требуется правка текста в WORKFLOW.md (простое исправление)

## Е. Неопределённость формата передачи контекста Парсер → Студия

**Проблема:** Переход между модулями использует `sessionStorage` для передачи `bookId`, `chapterId`, активной вкладки. Но:
1. Нет формального контракта — какие именно ключи sessionStorage обязательны для корректного открытия Студии?
2. Если пользователь открывает Студию по прямой ссылке (без прохода через Парсер) — какие данные отсутствуют и как это обрабатывается?
3. `useStudioSession` полагается на серверные данные (`book_scenes`, `scene_segments`), но что если пуш ещё не был сделан?

**Статус:** 🔴 Ожидает обсуждения

## Ж. Отсутствие единого реестра AI-ролей и их привязки к Edge Functions

**Проблема:** `src/config/aiRoles.ts` определяет роли (Переводчик, Режиссёр, Профайлер и т.д.), но:
1. Нет явного маппинга: какая роль → какая edge function → какой промпт
2. `ModelSelector` позволяет выбрать модель для роли, но edge functions не всегда получают этот выбор (часть хардкодит модель)
3. Нет документации: какие роли обязательны для какого этапа пайплайна
