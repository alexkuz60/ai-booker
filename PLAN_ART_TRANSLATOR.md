# Art Translator — План модуля художественного перевода

> Статус: **Фаза 1 — каркас** | Дата: 2026-03-28

## 1. Цель

Многоагентный пайплайн профессионального художественного перевода книги (RU→EN и обратно),
с объективной оценкой качества через многовекторное сравнение эмбеддингов —
**без необходимости знания целевого языка пользователем**.

## 2. Ключевые принципы

- **Ролевой подход** (паттерн «AI-Гидра»): каждый агент — специалист с чёткой зоной ответственности.
- **Фонетические эмбеддинги**: программная реализация для точного сравнения просодии и ритмики.
- **Измеримость без билингвальности**: радар-диаграмма эмбеддингов позволяет оценить качество перевода
  по объективным метрикам (семантика, тональность, ритмика, культурный код).
- **Сохранение метаданных**: SSML-разметка, BPM, психотипы персонажей, паузы — переносятся в перевод
  для бесшовной TTS-интеграции на целевом языке.
- **Сегменты Студии как вход**: раскадровка (storyboard) оригинала — готовая «партитура» с типами,
  спикерами и аннотациями, структурирующая задание для каждого агента.

## 3. Архитектура: Dual-Project

### 3.1. Принцип

Перевод хранится как **отдельный OPFS-проект** с идентичной иерархией ID.

```
МояКнига/                          ← оригинал
  project.json                     ← { language: "ru" }
  scene_index.json
  characters.json
  chapters/{chId}/scenes/{scId}/
    storyboard.json

МояКнига_EN/                       ← перевод
  project.json                     ← { language: "en",
                                        sourceProjectName: "МояКнига",
                                        targetLanguage: "en" }
  scene_index.json                 ← копия структуры оригинала
  characters.json                  ← переведённые имена/описания
  chapters/{chId}/scenes/{scId}/
    storyboard.json                ← переведённые фразы
```

### 3.2. Преимущества

- Все ID глав/сцен/сегментов **идентичны** → бесплатный маппинг «оригинал ↔ перевод».
- `ProjectStorage` переиспользуется as-is, `paths.*` работают без изменений.
- Push to Server — независимый для каждой версии.
- Оригинал остаётся **read-only** в контексте страницы перевода.

### 3.3. ProjectMeta (расширение)

```typescript
interface ProjectMeta {
  // ... existing fields ...
  sourceProjectName?: string;   // OPFS-имя проекта оригинала (только у переводов)
  targetLanguage?: "en" | "ru"; // целевой язык (только у переводов)
}
```

### 3.4. Создание проекта-перевода

1. Пользователь нажимает «Создать перевод» на странице `/translation`.
2. Копируется структура: `toc.json`, `scene_index.json`, `chapters/` (с пустыми storyboard).
3. `project.json` заполняется `sourceProjectName` + `targetLanguage`.
4. Персонажи копируются для последующего перевода имён/описаний.

## 4. Агентный пайплайн

### 4.1. Роли

| Роль | ID | Tier | Задача |
|------|----|------|--------|
| **Арт-переводчик** (Art Translator) | `art_translator` | standard | Дословный + литературный перевод сегментов |
| **Литературный редактор** (Literary Editor) | `literary_editor` | heavy | Стилистическая доводка, ритмика, культурная адаптация |
| **Критик перевода** (Translation Critic) | `translation_critic` | standard | Оценка качества по 5 осям Quality Radar |

> Системный `translator` (lite) остаётся для утилитарных переводов (поиск, промпты SFX/музыки).

### 4.2. Цикл обработки (на уровне сегмента)

```
Оригинал (сегмент из storyboard)
  │
  ├─→ [Переводчик] → подстрочник
  │
  ├─→ [Лит. редактор] → художественный перевод
  │       (получает: подстрочник + оригинал + контекст персонажа + SSML/BPM)
  │
  ├─→ [Эмбеддинг-валидация] → многовекторное сравнение
  │       (семантика, тональность, ритмика, фонетика, культурный код)
  │
  ├─→ [Критик] → анализ отклонений + рекомендации
  │       (получает: оригинал + перевод + дельты эмбеддингов)
  │
  └─→ Финальная версия (или повторный цикл редактор→критик)
```

### 4.3. Условие выхода из цикла

- Все векторы эмбеддингов выше порога (настраиваемого пользователем).
- Максимум 2 итерации редактор→критик (защита от зацикливания и расхода токенов).
- Пользователь может принять/отклонить на любом шаге.

## 5. Многовекторная оценка качества (Quality Radar)

### 5.1. Оси сравнения

| Ось | Метод | Что измеряет |
|-----|-------|-------------|
| **Семантика** | Gemini Embedding / Cohere Embed v4 | Смысловая близость |
| **Тональность** | Эмбеддинг + sentiment-анализ | Эмоциональный тон (радость/грусть/ирония) |
| **Ритмика** | Фонетические эмбеддинги (программные) | Темп, длина фраз, паузы, BPM-совместимость |
| **Фонетика** | Программная реализация | Звуковой рисунок, аллитерация, рифма |
| **Культурный код** | LLM-оценка (критик) | Идиомы, реалии, аллюзии |

### 5.2. UI: Радар качества перевода

- Пятиосевая радар-диаграмма (recharts) на уровне сегмента и агрегат на уровне сцены/главы.
- Цветовая индикация: зелёный (>0.85), жёлтый (0.70–0.85), красный (<0.70).
- Клик по оси → детализация: конкретные фразы с наибольшим отклонением.
- Агрегированный «балл перевода» (взвешенное среднее) на уровне книги.

## 6. UI: Страница `/translation`

### 6.1. Layout

```
┌─────────────────────────┬─────────────────────────┐
│   Левая панель (50%)    │   Правая панель (50%)   │
│                         │                         │
│  Навигатор глав         │  Quality Radar          │
│  (ChapterNavigator,     │  Синонимы / варианты    │
│   статусы: pending →    │  Критическая оценка     │
│   translated → reviewed)│  Детализация по осям    │
│                         │                         │
│  Раскадровка            │                         │
│  (билингва-аккордеоны)  │                         │
└─────────────────────────┴─────────────────────────┘
```

### 6.2. Билингва-аккордеоны

На уровне каждого сегмента:

```
▶ [dialogue] Анна (RU)     ← оригинал (свёрнут по умолчанию, read-only)
▼ [dialogue] Anna (EN)     ← перевод (развёрнут, редактируемый)
```

- Дефолт: оригинал свёрнут, перевод развёрнут (фокус на работе).
- Кнопка «Toggle all» для быстрого переключения режима просмотра.
- Оригинал — strictly read-only, визуально приглушён.

### 6.3. Навигатор глав

Переиспользуется `ChapterNavigator` из Студии с адаптированными статусами:

| Статус Студии | Статус Перевода |
|---------------|-----------------|
| pending | pending |
| segmented | translated |
| rendered | reviewed |

## 7. Интеграция с существующей архитектурой

### 7.1. AI-роли (aiRoles.ts) ✅

Добавлены 3 новые роли в `src/config/aiRoles.ts`:
- `art_translator` (standard, poolable) — дословный + литературный перевод
- `literary_editor` (heavy, poolable) — стилистическая доводка
- `translation_critic` (standard, poolable) — Quality Radar оценка

Системный `translator` (lite) не затронут — остаётся для утилитарных переводов.
Иконки и цвета добавлены в `RoleBadge.tsx`.

### 7.2. Edge Functions

| Функция | Роль | Вход | Выход |
|---------|------|------|-------|
| `translate-literal` | translator | сегмент + контекст | подстрочник |
| `translate-literary` | literary_editor | оригинал + подстрочник + мета | худ. перевод |
| `critique-translation` | translation_critic | оригинал + перевод + эмбеддинг-дельты | анализ + рекомендации |
| `compute-embeddings` | — | текст(ы) | векторы по осям |

### 7.3. Провайдеры эмбеддингов

- **Gemini Embedding** (`text-embedding-004`) — через Lovable AI (admin) или API key.
- **Cohere Embed v4** — через API key (альтернатива).
- Фонетические эмбеддинги — **программная реализация** (клиентская, без внешнего API).

### 7.4. Хранение

- Параллельные проекты в OPFS: `{Title}/` и `{Title}_{lang}/`.
- Эмбеддинги кэшируются локально (OPFS) для повторного сравнения.
- Метаданные перевода (score, iteration count, агент) — в `translation_meta.json` внутри сцены.

## 8. Ограничения и риски

- **Стоимость**: Heavy-tier модели для редактора и критика; эмбеддинги — дополнительные вызовы.
  Митигация: батчинг сегментов, кэширование эмбеддингов, лимит итераций.
- **Детерминизм**: LLM-перевод недетерминирован.
  Митигация: фиксация seed/temperature где возможно; версионирование результатов.
- **Поэзия/стихи**: Особый режим с приоритетом фонетической оси.
