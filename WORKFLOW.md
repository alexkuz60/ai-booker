# AI-Booker: Рабочий процесс создания аудиокниги

> Документ описывает полный пайплайн — от загрузки файла до подготовки к озвучке.  
> Актуальная дата: 2026-03-09. Статус каждого шага: ✅ реализовано, 🔧 частично, ⬜ запланировано.

---

## 1. Загрузка книги ✅

**Модуль:** Парсер → Библиотека → Загрузка

| Параметр | Детали |
|----------|--------|
| Формат файла | PDF (извлечение текста + оглавления через `pdfjs-dist`) |
| Хранение файлов | Приватный бакет `book-uploads` в облачном хранилище, привязка к `user_id` |
| Повторная загрузка | Если файл с тем же именем уже существует — старая версия автоматически удаляется из хранилища и из БД (книга, части, главы, сцены), после чего загружается новая |
| Ограничения | Максимальный размер PDF не ограничен жёстко на фронтенде; обработка выполняется целиком в браузере |

**Порядок действий пользователя:**
1. Открыть модуль **Парсер**.
2. Нажать **«Загрузить книгу»** или выбрать существующую в библиотеке.
3. Выбрать PDF-файл — начнётся автоматическое извлечение оглавления.

## 2. Извлечение структуры (TOC) ✅

**Модуль:** Парсер (автоматический этап)

Сразу после загрузки PDF система:
1. Извлекает встроенное оглавление (`pdf.getOutline()`) — иерархически, с определением уровней вложенности.
2. Вычисляет диапазоны страниц для каждой записи (по позициям следующих записей).
3. Классифицирует каждую запись по типу раздела:
   - **📖 Содержание** — основные главы и подглавы
   - **📝 Предисловие** — введение, пролог, от автора
   - **📜 Послесловие** — эпилог, заключение
   - **🔗 Примечания** — сноски, библиография, глоссарий
   - **📎 Приложения** — дополнительные материалы
4. Нормализует уровни вложенности (устраняет «осиротевшие» подпункты).
5. Сохраняет структуру в БД: `books → book_parts → book_chapters`.

## 3. Навигатор структуры книги ✅

**Модуль:** Парсер → Рабочая область (левая панель)

Навигатор отображает полную структуру книги в виде дерева с возможностью ручного редактирования:

### Отображение
- Древовидная структура с группировкой по частям (Part I, Part II…)
- Раскрываемые узлы для частей и глав с подглавами
- Разделение на секции: содержание, предисловие, послесловие, примечания, приложения
- Индикаторы статуса каждой главы: ⏳ ожидает / ⚙️ анализ / ✅ готово / ❌ ошибка
- Ссылки на страницы PDF (клик открывает PDF на нужной странице)

### Инструменты редактирования
| Инструмент | Описание |
|------------|----------|
| **Inline-редактирование заголовка** | Двойной клик по заголовку главы → ввод нового названия |
| **Редактирование начальной страницы** | Двойной клик по номеру страницы → изменение (автоматически корректируется конечная страница предыдущей главы) |
| **Изменение уровня вложенности** | Кнопки `<` (outdent) и `>` (indent) — рекурсивно смещают дочерние элементы |
| **Удаление записей** | Кнопка 🗑️ — удаляет запись и все дочерние элементы (с подтверждением) |
| **Переименование частей** | Двойной клик по заголовку части |
| **Открытие PDF** | Кнопка 🔗 — открывает PDF в новой вкладке на нужной странице |
| **Мультивыбор** | `Ctrl+Click` (добавить/убрать), `Shift+Click` (диапазон) — массовые операции |

Все изменения автоматически синхронизируются с БД.

## 4. Семантический анализ глав ✅

**Модуль:** Парсер → Рабочая область (правая панель) + Edge Function `parse-book-structure`

### Процесс (двухэтапный)

**Этап 1 — Определение границ сцен:**
1. Извлечение текста из PDF по диапазону страниц главы (с обработкой сносок и переносов).
2. Отправка текста в AI с режимом `boundaries` — определение начальных строк каждой сцены.
3. Разделение текста по маркерам на отдельные сцены.

**Этап 2 — Обогащение метаданными:**
1. Для каждой сцены AI определяет (режим `enrich`):
   - **Тип сцены**: action, dialogue, lyrical_digression, description, inner_monologue, mixed
   - **Настроение** (mood): тональная характеристика
   - **BPM**: темп сцены (числовое значение)
   - **Заголовок**: краткое название сцены
2. Результаты сохраняются в таблицу `book_scenes`.

### Выбор AI-модели
Пользователь может выбрать модель для анализа из реестра:
- **Встроенные** (без ключа): Gemini 3 Flash/Pro, Gemini 2.5 Pro/Flash/Lite, GPT-5/Mini/Nano/5.2
- **ProxyAPI** (нужен ключ): Claude 4 Sonnet/Opus, GPT-5/5-mini
- **OpenRouter** (нужен ключ): любая модель из каталога

### UX анализа
- Прогресс-лог в реальном времени (определение границ → расчёт страниц → настроение/темп для каждой сцены)
- Фоновый предвыбор (prefetch) до 3 следующих глав
- Возможность повторного/продолжения анализа
- Карточки сцен с цветовой кодировкой по типу, длительностью и раскрываемым превью текста

## 5. Передача главы в Студию ✅

**Модуль:** Парсер → Студия

### Условия доступности
Кнопка переноса (🎬) отображается когда:
- Глава верхнего уровня: сама глава и **все** её дочерние подглавы имеют статус `done` с непустыми сценами.
- Подглава: аналогичная рекурсивная проверка.

### Механизм передачи
1. Собираются сцены из самой главы и всех её дочерних подглав.
2. Для вложенных элементов заголовок формируется как: `«Название родителя (Часть N)»`, где N — порядковый номер.
3. Данные сохраняются в `sessionStorage` (с автоматической деградацией при переполнении — обрезка текста сцен).
4. Пользователь перенаправляется на страницу Студии.

## 6. Студия: навигатор сцен ✅

**Модуль:** Студия (левая панель)

- Отображение списка сцен загруженной главы.
- Цветовая кодировка по типу сцены (action, dialogue, description…).
- Оценка длительности каждой сцены и главы в целом (по количеству символов).
- Статистика в хедере: общая длительность главы, количество символов.

### Индикаторы статуса аудио ✅
Каждая сцена в навигаторе отображает иконку готовности:
- 🎬 **Film** — нет аудио (только раскадровка).
- 🔈 **Контурный динамик** — частично отрендерено (есть аудио, но не для всех сегментов).
- 🔊 **Сплошной динамик** — полностью синтезирована (все сегменты имеют аудио).
- ⚠️ **Жёлтый треугольник** — аудио устарело (параметры голоса персонажа изменены после синтеза).

### Пакетный ре-синтез ✅
Если хотя бы одна сцена помечена как «устаревшая» (⚠️), в хедере навигатора появляется кнопка пакетного ре-синтеза. При нажатии:
1. Автоматически находятся все сцены с устаревшим аудио.
2. Последовательно запускается ре-синтез каждой сцены.
3. Прогресс отображается в реальном времени.

## 7. Раскадровка (Storyboard) ✅

**Модуль:** Студия → вкладка «Раскадровка» + Edge Function `segment-scene`

### Автоматическая сегментация
При выборе сцены в навигаторе система автоматически:
1. Проверяет, есть ли уже сегменты в БД для данной сцены.
2. Если нет — запускает AI-анализ, разбивающий текст сцены на типизированные сегменты.

### Типы сегментов
| Тип | Описание |
|-----|----------|
| 📖 **Рассказчик** (narrator) | Повествование от третьего лица |
| 👤 **От первого лица** (first_person) | Повествование от первого лица |
| 💬 **Диалог** (dialogue) | Прямая речь персонажа (с указанием спикера) |
| 🧠 **Мысли** (inner_thought) | Внутренний монолог |
| ✍️ **Эпиграф** (epigraph) | Цитата или эпиграф |
| 🎵 **Лирика** (lyric) | Стихотворный или песенный фрагмент |
| 📝 **Сноска** (footnote) | Примечание |

### Структура данных
Каждый сегмент разбивается на **фразы** (предложения) — атомарные единицы для озвучки:
- `scene_segments` → тип, спикер, порядковый номер, metadata (jsonb)
- `segment_phrases` → текст фразы, порядковый номер

### Редактирование
- Каждая фраза редактируемая inline (клик по иконке ✏️ → textarea → Enter для сохранения, Esc для отмены).
- Изменения сохраняются в БД мгновенно.
- Возможность переанализа сцены (кнопка «Переанализ»).

### Интерактивные бейджи сегментов ✅
- **Тип блока** (narrator, dialogue, first_person…) — кликабельный бейдж-селектор. Позволяет вручную исправить ошибку классификации AI. Выбор сохраняется в БД.
- **Персонаж** — кликабельный бейдж-селектор. Если спикер не назначен, отображается «персонаж ?» (оранжевый). Выбор доступен из списка персонажей книги.
- **Связка тип ↔ персонаж**: при назначении персонажа на блок определённого типа (narrator, first_person, inner_thought, epigraph, lyric, footnote) — все остальные блоки того же типа в пределах сцены автоматически получают того же персонажа. Для **dialogue** смена применяется строго индивидуально.
- **Пропагация типов narrator↔first_person**: при смене типа одного блока с `narrator` на `first_person` (или обратно) — **все** блоки старого типа в сцене автоматически меняются на новый. После пропагации запускается `syncSceneCharacters`, удаляющий персонажей без оставшихся сегментов → пустые треки на таймлайне исчезают.
- Связка работает в пределах одной сцены и синхронизирована с табом «Персонажи» и таймлайном.

### Детекция инлайн-вставок (Inline Narrations) ✅

**Edge Function:** `detect-inline-narrations`

В русской литературе внутри прямой речи часто встречаются авторские ремарки:
> «Родя, — *тихо позвал он,* — ты не умирай»

Кнопка **«Вставки»** (иконка ScanSearch) в тулбаре Раскадровки запускает AI-детекцию:

1. Загружает все диалоговые сегменты сцены.
2. AI (Gemini 2.5 Flash) анализирует каждый сегмент и выделяет:
   - **inline_narrations** — массив `{ text, insert_after }` (текст вставки и позиция).
   - **clean_text** — текст диалога с удалёнными авторскими словами.
3. Найденные вставки сохраняются в `metadata.inline_narrations` сегмента.
4. Оригинальные фразы очищаются от авторского текста (предотвращает дублирование при синтезе).

**Отображение:**
- В заголовке сегмента — иконка 💬 с количеством вставок.
- Под текстом фраз — бледно-жёлтый блок в формате: `после [фраза] → [вставка]`.

### Индикация аудио в сегментах ✅
Каждый сегмент отображает статус аудио и кнопку управления:
- ✅ **Готово** — зелёная галочка + длительность (`2.5s`).
- ▶ **Нет аудио** — кнопка ручного синтеза отдельного сегмента.
- 🔄 **Ре-синтез** — кнопка принудительного обновления существующего аудио.
- ❌ **Ошибка** — красный крестик.

### Синхронизация с другими модулями ✅
- При смене/назначении спикера персонаж автоматически добавляется в `character_appearances` (→ дорожка на таймлайне).
- При удалении спикера — персонаж убирается из списка появлений, если у него не осталось сегментов.
- Двусторонняя синхронизация с таймлайном: двойной клик по клипу → выделение сегмента в раскадровке и наоборот.

## 8. Профилирование персонажей ✅

**Модуль:** Студия → вкладка «Персонажи» + Edge Function `profile-characters`

### Автоматическое извлечение
Во время сегментации сцены (шаг 7) система автоматически:
1. Извлекает всех спикеров из диалоговых сегментов.
2. Создаёт/обновляет записи в `book_characters` (upsert по имени).
3. Создаёт записи в `character_appearances` (связь персонаж ↔ сцена ↔ сегменты).
4. Сопоставляет новых спикеров с существующими по имени и алиасам (регистронезависимо) — предотвращает дублирование.

### AI-профилирование
По запросу (кнопка «AI-Профайлинг») или автоматически при подборе актёров:
1. Собирает реплики каждого персонажа из сегментов.
2. Собирает фрагменты нарратора для контекста.
3. Отправляет в AI (Gemini 2.5 Flash) для определения:
   - **Пол** (male / female / unknown)
   - **Возрастная группа** (child / teen / young / adult / elder)
   - **Темперамент** (sanguine / choleric / melancholic / phlegmatic / mixed)
   - **Стиль речи** (2–3 предложения)
   - **Психологический портрет** (3–5 предложений)
   - **Алиасы** (все варианты имени, прозвища, обращения)

### Инкрементальное профилирование ✅
При загрузке **новой главы** в студию:
1. Новые персонажи → создаётся полный профайл.
2. Известные персонажи, появляющиеся в новых сценах → профайл обновляется только при существенных новых проявлениях характера.
3. Известные персонажи, не появляющиеся в новых сценах → профайл не изменяется.

### Ручное редактирование
Пользователь может вручную изменить:
- Пол (через popover-меню)
- Возрастную группу (через popover-меню)
- Темперамент (через popover-меню)

### Линковка «От первого лица» ✅
Тип сегмента **first_person** автоматически связывается с назначенным персонажем через `scene_type_mappings`. Это обеспечивает:
- Корректный голос при синтезе повествования от первого лица.
- Голос для инлайн-вставок (если есть first_person → его голос вместо Рассказчика).
- Единообразное назначение: при выборе персонажа для одного блока first_person — все блоки того же типа в сцене обновляются.

### Слияние и очистка дубликатов ✅
- **Ручное слияние**: мультивыбор персонажей → кнопка «Слить» → один персонаж поглощает другого (алиасы, появления, сегменты переносятся).
- **Автоочистка дубликатов**: автоматический поиск пар с совпадением имён/алиасов → объединение с переносом всех связей (`character_appearances`).
- **Пометка «Экстра»**: второстепенные персонажи можно пометить как extras (сортировка в конец списка).

Кнопка **«Подбор Актёров»** в хедере вкладок запускает двухэтапный процесс:

**Этап 1 — Кастинг (назначение голосов):**

  - Мужчины: kirill, alexander, zahar, ermil, anton, filipp, madirus
  - Дети: masha (ж), filipp (м)
  - Пожилые: omazh/jane (ж), zahar (м)
- Избегание дублирования голосов между персонажами одного пола.
- На основе темперамента назначается **роль** (интонационный стиль): neutral, friendly, strict, good, evil, whisper.

**Этап 2 — Инкрементальное профилирование:**
- Автоматически запускается после кастинга для уточнения профилей на основе текущих сцен.

### Мульти-провайдер голосов ✅
Настройки голоса разделены на два таба по провайдерам:

#### Таб «Yandex SpeechKit»
| Параметр | Описание |
|----------|----------|
| Голос | Выбор из реестра Yandex SpeechKit (15+ голосов) |
| Роль | Интонационный стиль (зависит от доступных ролей голоса) |
| Скорость | 0.1 – 3.0 (по умолчанию 1.0) |
| Громкость | Нормализация для балансировки голосов |
| Высота | Тонкая настройка тембра |

#### Таб «ElevenLabs» 🔧
| Параметр | Описание |
|----------|----------|
| Голос | Выбор из реестра ElevenLabs (18+ голосов, мультиязычные) |
| Stability | 0–1 — стабильность интонации (по умолчанию 0.6) |
| Similarity Boost | 0–1 — близость к эталонному голосу (по умолчанию 0.75) |
| Style Exaggeration | 0–1 — выразительность стиля (по умолчанию 0.4) |
| Speed | 0.5–2.0 — скорость речи (по умолчанию 0.95) |
| Кредиты | Индикатор остатка символов на тарифе (автозагрузка из API) |

> **Требования:** для ElevenLabs необходим API-ключ (платный план), указанный в Профиле → API-ключи.

### Предпрослушивание ✅
Кнопка ▶ рядом с настройками голоса — синтезирует короткий отрывок через Edge Function `yandex-tts` или `elevenlabs-tts` (в зависимости от активного таба) и воспроизводит в браузере.

## 10. Таймлайн ✅

**Модуль:** Студия (нижняя панель)

### Режим «Сцена»
Детальный мультитрековый режим для работы с отдельной сценой:
- Линейка времени с масштабированием (zoom in/out, fit to width).
- Динамические дорожки: по одной на каждого персонажа сцены (из `character_appearances`) + «Рассказчик» (авто-fallback) + Атмосфера + SFX.
- Реальные клипы из сегментов с цветовой кодировкой персонажей.
- Визуальное различие: озвученные клипы (🔊, сплошная заливка) vs оценочные (полупрозрачные, штриховка).
- Длительность клипов синхронизируется с реальными данными из `segment_audio` (fallback — оценка по символам).
- Двусторонняя синхронизация: клик по дорожке ↔ выбор персонажа в панели настроек.
- **Двусторонняя синхронизация с Раскадровкой:** двойной клик по клипу → выделение сегмента в Storyboard и наоборот.
- **Overlay-клипы:** инлайн-вставки рассказчика отображаются как отдельные клипы поверх дорожки диалога со смещением по времени.
- Клипы сегментов, находящихся в процессе синтеза, визуально выделяются (анимация).

### Режим «Глава»
Обзорный режим для всей главы:
- Единая дорожка «Сцены» с одним клипом на каждую сцену.
- Цветовая кодировка сцен (чередующаяся палитра).
- Отображение длительности каждой сцены (суммарная из сегментов или оценочная).
- Текущая сцена визуально выделяется.
- **Двойной клик** по клипу сцены → переход в режим «Сцена» с выбором этой сцены в навигаторе.

### Транспорт и воспроизведение ✅
- Кнопки Play / Pause / Stop.
- Плейхед (маркер текущей позиции) — отображается всегда, движется в реальном времени при воспроизведении.
- **Seek по клику** — клик в любой точке шкалы перемещает плейхед и продолжает воспроизведение с новой позиции.
- Последовательное воспроизведение клипов с готовым аудио (из `segment_audio`).
- **Overlay-воспроизведение**: инлайн-вставки рассказчика проигрываются параллельно основному треку в рассчитанной временной позиции.
- **Sample-accurate синхронизация**: Tone.Transport как единый источник времени, корректный расчёт оффсетов для seek mid-clip.

- Регулировка громкости (персистентная в localStorage).
- Счётчик времени: текущая позиция / общая длительность.

### Микшер (Mixer) ✅

**Расположение:** Сайдбар таймлайна (компонент `TrackMixerStrip`)

| Элемент | Описание |
|---------|----------|
| **VU-слайдер** | Canvas-компонент с реальным VU-метрингом (~30fps), яркими контурами и красной меткой на ручке |
| **Громкость** | Per-track volume слайдер (-∞ … +6 dB) |
| **Панорама** | Per-track pan (-1 … +1) |
| **Mute** | Заглушение трека (совместимо с Solo) |
| **Solo** | Солирование — авто-мьют остальных треков |
| **Pre-FX** | Компрессор per-track с байпассом (ratio=1 = bypass) |
| **Insert Reverb** | Per-track реверб с wet-контролем и байпассом |

**Режимы отображения:**
- **Развёрнутый** (360px): колоночная вёрстка с заголовками «Уровень» / «Панорама», названия треков (90px)
- **Свёрнутый**: только имя + цветная точка персонажа

**Персистентность:** гибридная стратегия per scene:
1. **localStorage** — мгновенное восстановление при переключении сцен (без задержки UI)
2. **Cloud sync** — фоновая запись в `user_settings` с debounce 300ms для кросс-сессионного сохранения

### Автопересчёт после синтеза ✅
После завершения синтеза или ре-синтеза:
- Позиции всех последующих клипов автоматически пересчитываются.
- Общая шкала таймлайна обновляется.
- Предотвращается наложение звуковых дорожек.

### Layout
- Drag-resize панели по вертикали.
- Сворачивание/разворачивание.
- Персистентность состояния (размер, collapse) в localStorage.

**⬜ Запланировано:** drag-and-drop клипов, fade-in/fade-out, экспорт.

## 10a. Синтез сцены (TTS) ✅

**Модуль:** Студия → Раскадровка → кнопка «Синтез сцены» + Edge Function `synthesize-scene`

### Процесс
1. Нажать кнопку **«Синтез сцены»** в хедере Раскадровки.
2. Для каждого сегмента:
   - Текст всех фраз склеивается в единый блок.
   - Определяется голос/роль/скорость из `voice_config` назначенного персонажа.
   - Вызывается Edge Function `yandex-tts` (Yandex SpeechKit).
   - Аудиофайл сохраняется в бакет `user-media` (путь: `tts/{userId}/{sceneId}/{segmentId}.mp3`).
   - Метаданные записываются в таблицу `segment_audio` (длительность, путь, статус, voice_config).
3. После завершения таймлайн автоматически обновляется — клипы отображаются с реальной длительностью.

### Смарт-разбивка больших блоков ✅
Yandex SpeechKit v3 имеет лимит ~240 символов на один вызов. Для длинных сегментов:
1. Текст автоматически разбивается по границам предложений (splitTextForV3).
2. Каждый фрагмент синтезируется отдельно.
3. Результирующие аудиобуферы конкатенируются в единый MP3-файл.
4. Валидация: пустые буферы отбраковываются перед сохранением.
> Для v1 лимит значительно выше, разбивка не требуется. SSML-контент всегда отправляется через v1.

### Кэширование и инвалидация ✅
- `segment_audio` служит кэшем: при повторном синтезе система сравнивает текущий хеш текста (FNV-1a 32-bit) и параметры голоса с сохранёнными.
- Если текст или голос не изменились — сегмент пропускается (используется кэш).
- Если текст или голос изменились — аудио перегенерируется.

### Ре-синтез ✅
Доступен на трёх уровнях:
- **Индивидуальный** — кнопка 🔄 на каждом сегменте в Раскадровке (принудительный ре-синтез).
- **Пакетный** — кнопка ▶ на сегментах без аудио (ручная доозвучка пропущенных).
- **Батч по главе** — кнопка в навигаторе для ре-синтеза всех устаревших сцен (когда voice_config персонажа изменился).

### Обработка инлайн-вставок ✅
Для диалоговых сегментов с `metadata.inline_narrations`:

**v1 (SSML-совместимые голоса):**
- Авторская ремарка инжектируется как SSML-пауза в точке вставки.
- Основной диалог синтезируется единым потоком с паузами.

**v3 (голоса без SSML):**
- Диалог синтезируется чистым текстом (без пауз).
- Авторские ремарки синтезируются отдельно голосом Рассказчика.
- На таймлайне вставки размещаются как overlay-клипы со смещением по времени.

### Выбор голоса для вставок ✅
Динамическая логика:
- Если в сцене есть сегмент **«От первого лица»** с назначенным персонажем → его голос.
- Иначе → глобальный голос **«Рассказчика»** книги.

### Ограничения
- Доступно только администраторам (аналогично Yandex TTS).

## 11. The Atmosphere Engine 🔧

**Модуль:** Студия → вкладка «Атмосфера» + Edge Functions `elevenlabs-sfx`, `elevenlabs-music`, `generate-atmosphere-prompt` (⬜)

### Текущее состояние (ручной режим) ✅

Вкладка «Атмосфера» в Студии позволяет вручную генерировать звуковое оформление по текстовому описанию:

| Под-вкладка | Провайдер | Настройки |
|-------------|-----------|-----------|
| **Sound Effects** | ElevenLabs Sound Generation | Длительность (1–22 сек), Prompt Influence (0–100%) |
| **Ambience** | ElevenLabs Sound Generation | Длительность (1–22 сек), Prompt Influence (0–100%) |
| **Music** | ElevenLabs Music | Длительность (1–120 сек) |

- Встроенный плеер для предпрослушивания.
- Сохранение в `user-media` storage (путь: `{user_id}/{category}/{slug}-{timestamp}.mp3`).
- Абстракция `SoundProvider` (`src/lib/soundProvider.ts`) позволяет подключать других провайдеров.

### Запланировано: автогенерация по метаданным сцены ⬜

**Этап 1 — AI Prompt Builder:**
1. Edge-функция `generate-atmosphere-prompt` принимает метаданные сцены: `mood`, `scene_type`, `bpm`, краткое содержание.
2. AI (Gemini 3 Flash) генерирует 2–3 звуковых промпта:
   - **Ambience layer** — фоновый звук окружения (лес, город, дождь…)
   - **Music layer** (опционально) — фоновая музыка, соответствующая настроению
   - **SFX accents** (опционально) — точечные звуковые эффекты
3. Каждый промпт содержит: текст описания, рекомендуемую длительность, громкость, fade-параметры.

**Этап 2 — Auto-Atmosphere:**
1. Кнопка «🎭 Auto-Atmosphere» в `AtmospherePanel` (или `StoryboardPanel`).
2. Загрузка метаданных текущей сцены из `book_scenes`.
3. Вызов Prompt Builder → параллельная генерация слоёв через `generateSound()`.
4. Автосохранение в `user-media` + запись в `scene_atmospheres`.

**Этап 3 — Таблица `scene_atmospheres`:**

| Поле | Тип | Описание |
|------|------|----------|
| `id` | uuid PK | — |
| `scene_id` | uuid FK → book_scenes | Привязка к сцене |
| `layer_type` | text | ambience / music / sfx |
| `audio_path` | text | Путь в storage |
| `prompt_used` | text | Промпт, по которому сгенерировано |
| `duration_ms` | int | Длительность аудио |
| `volume` | float | Рекомендуемая громкость (0–1) |
| `fade_in_ms` | int | Длительность fade-in |
| `fade_out_ms` | int | Длительность fade-out |
| `created_at` | timestamptz | — |

**Этап 4 — Timeline-интеграция:**
- `useTimelineClips` подтягивает записи из `scene_atmospheres` как клипы на треки `atmosphere` / `sfx`.
- Если атмосферный клип короче длительности сцены — автоматическое зацикливание (loop) с crossfade.
- Поддержка drag & drop для ручного позиционирования (после реализации интерактивного таймлайна).

**Этап 5 — Batch-режим:**
- Кнопка «Сгенерировать атмосферу для всей главы» в навигаторе.
- Итерация по сценам с отображением прогресс-бара.
- Пропуск сцен, у которых уже есть `scene_atmospheres`.

## 11. Аудио-движок (AudioEngine) ✅

**Модуль:** `src/lib/audioEngine.ts` — синглтон на базе Tone.js

### Архитектура сигнальной цепочки

```
Трековая цепочка (per-track):
  Player (моно TTS) → PreFX (Compressor, bypass) → Channel (Vol/Pan) → Reverb (bypass) → Split L/R Meter → Bus
                                                          └→ MeterMono

Суб-шины:
  VoiceBus ──┐
  AtmoBus  ──┼→ MasterBus
  SfxBus   ──┘

Мастер-цепочка:
  MasterBus → EQ3 → Compressor → Limiter → Filter[1..5] → MBC (3-Band) → Reverb → Split L/R → Meters + FFT → Destination
```

### Трековый уровень

| Компонент | Реализация | Статус |
|-----------|-----------|--------|
| Transport | `Tone.Transport` — единый источник истины | ✅ |
| Per-track Vol/Pan | `Tone.Channel` | ✅ |
| Pre-FX контейнер | `Tone.Compressor` с байпассом (ratio=1) | ✅ |
| Per-track Reverb | `Tone.Reverb` с wet-контролем и байпассом | ✅ |
| Per-track Metering | `Tone.Meter` (моно) + `Tone.Split` → 2×`Tone.Meter` (L/R) | ✅ |
| 3 суб-шины | Voice / Atmosphere / SFX → MasterBus | ✅ |
| Mute / Solo | Per-track, с авто-мьют при соло | ✅ |
| Fade-in / Fade-out | Через автоматизацию Tone.js | ⬜ |
| Envelope Editor | Визуальный редактор огибающих | ⬜ |

### Мастер-цепочка обработки (Mastering Chain) ✅

**Порядок обработки:** EQ3 → Compressor → Limiter → 5-Band Filter → 3-Band MBC → Reverb

| Эффект | Группа | Реализация | UI |
|--------|--------|-----------|-----|
| **EQ3** | Pre | `Tone.EQ3` (low/mid/high ±12 dB) | Слайдеры |
| **Compressor** | Pre | `Tone.Compressor` | Threshold, Ratio, Attack, Release, Knee |
| **Limiter** | Post | `Tone.Limiter` | Threshold, Release |
| **5-Band Filter** | Master | 5× `Tone.BiquadFilter` (HP, LS, PK, HS, LP) | Canvas-график: draggable узлы, freq/gain/Q |
| **MBC 3-Band** | Master | `Tone.MultibandCompressor` | Canvas-график: transfer curves, draggable кроссоверы |
| **Reverb** | Master | `Tone.Reverb` | Decay (0.1–10s), Wet (0–1) |

### Мастер-метеринг ✅

| Элемент | Описание |
|---------|----------|
| **Стерео VU L/R** | -96 dB … +3 dB, peak hold 1.5s, красная зона клиппинга |
| **Спектроанализатор** | FFT: 3 режима (bars / line / mirror), smoothing |
| **Плагин-стрип** | Bypass-кнопки с группировкой Pre / Post / Master |
| **Master BYP** | Глобальный байпасс всей цепи |

### Табовый интерфейс ✅

**Компонент:** `MasterEffectsTabs` — порядок табов: FFT → EQ → CMP → LIM → FLT → MBC → REV

Каждый таб: UI управления + inline bypass (ON/OFF) + disabled при bypass.

### UI микшера

| Элемент | Расположение | Статус |
|---------|-------------|--------|
| VU-слайдер с метром | Компонент `VuSlider` (Canvas + rAF ~30fps) | ✅ |
| Микшер-стрип | `TrackMixerStrip` — сворачиваемый, в сайдбаре таймлайна | ✅ |
| Развёрнутый режим (360px) | Vol + Pan слайдеры с VU, бейджи FX / RV | ✅ |
| Свёрнутый режим | Только имя + цветная точка | ✅ |
| Master L/R панель | `MasterMeterPanel` — стерео VU + плагин-стрип | ✅ |
| Мастер-эффекты | `MasterEffectsTabs` — 7 табов обработки | ✅ |

### Персистентность
- **Микшер:** localStorage (мгновенно) + cloud user_settings (debounce 300ms) per scene
- **Плагин bypass:** localStorage (`master-plugins-state`)
- **Активный таб:** localStorage (`master-fx-tab`)
