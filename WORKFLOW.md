# AI-Booker: Рабочий процесс создания аудиокниги

> Документ описывает полный пайплайн — от загрузки файла до подготовки к озвучке.  
> Актуальная дата: 2026-03-06. Статус каждого шага: ✅ реализовано, 🔧 частично, ⬜ запланировано.

---

## 1. Загрузка книги ✅

**Модуль:** Парсер → Библиотека → Загрузка

| Параметр | Детали |
|----------|--------|
| Формат файла | PDF (извлечение текста + оглавления через `pdfjs-dist`) |
| Хранение файлов | Приватный бакет `book-uploads` в облачном хранилище, привязка к `user_id` |
| Повторная загрузка | Если файл с тем же именем уже существует — старая версия автоматически удаляется из хранилища и из БД (книга, части, главы, сцены), после чего загружается новая |
| Ограничения | Максимальный размер PDF не ограничен жёстко на фронтенде; обработка выполняется целиком в браузере |

**Порядок действий пользователя:**
1. Открыть модуль **Парсер**.
2. Нажать **«Загрузить книгу»** или выбрать существующую в библиотеке.
3. Выбрать PDF-файл — начнётся автоматическое извлечение оглавления.

## 2. Извлечение структуры (TOC) ✅

**Модуль:** Парсер (автоматический этап)

Сразу после загрузки PDF система:
1. Извлекает встроенное оглавление (`pdf.getOutline()`) — иерархически, с определением уровней вложенности.
2. Вычисляет диапазоны страниц для каждой записи (по позициям следующих записей).
3. Классифицирует каждую запись по типу раздела:
   - **📖 Содержание** — основные главы и подглавы
   - **📝 Предисловие** — введение, пролог, от автора
   - **📜 Послесловие** — эпилог, заключение
   - **🔗 Примечания** — сноски, библиография, глоссарий
   - **📎 Приложения** — дополнительные материалы
4. Нормализует уровни вложенности (устраняет «осиротевшие» подпункты).
5. Сохраняет структуру в БД: `books → book_parts → book_chapters`.

## 3. Навигатор структуры книги ✅

**Модуль:** Парсер → Рабочая область (левая панель)

Навигатор отображает полную структуру книги в виде дерева с возможностью ручного редактирования:

### Отображение
- Древовидная структура с группировкой по частям (Part I, Part II…)
- Раскрываемые узлы для частей и глав с подглавами
- Разделение на секции: содержание, предисловие, послесловие, примечания, приложения
- Индикаторы статуса каждой главы: ⏳ ожидает / ⚙️ анализ / ✅ готово / ❌ ошибка
- Ссылки на страницы PDF (клик открывает PDF на нужной странице)

### Инструменты редактирования
| Инструмент | Описание |
|------------|----------|
| **Inline-редактирование заголовка** | Двойной клик по заголовку главы → ввод нового названия |
| **Редактирование начальной страницы** | Двойной клик по номеру страницы → изменение (автоматически корректируется конечная страница предыдущей главы) |
| **Изменение уровня вложенности** | Кнопки `<` (outdent) и `>` (indent) — рекурсивно смещают дочерние элементы |
| **Удаление записей** | Кнопка 🗑️ — удаляет запись и все дочерние элементы (с подтверждением) |
| **Переименование частей** | Двойной клик по заголовку части |
| **Открытие PDF** | Кнопка 🔗 — открывает PDF в новой вкладке на нужной странице |
| **Мультивыбор** | `Ctrl+Click` (добавить/убрать), `Shift+Click` (диапазон) — массовые операции |

Все изменения автоматически синхронизируются с БД.

## 4. Семантический анализ глав ✅

**Модуль:** Парсер → Рабочая область (правая панель) + Edge Function `parse-book-structure`

### Процесс (двухэтапный)

**Этап 1 — Определение границ сцен:**
1. Извлечение текста из PDF по диапазону страниц главы (с обработкой сносок и переносов).
2. Отправка текста в AI с режимом `boundaries` — определение начальных строк каждой сцены.
3. Разделение текста по маркерам на отдельные сцены.

**Этап 2 — Обогащение метаданными:**
1. Для каждой сцены AI определяет (режим `enrich`):
   - **Тип сцены**: action, dialogue, lyrical_digression, description, inner_monologue, mixed
   - **Настроение** (mood): тональная характеристика
   - **BPM**: темп сцены (числовое значение)
   - **Заголовок**: краткое название сцены
2. Результаты сохраняются в таблицу `book_scenes`.

### Выбор AI-модели
Пользователь может выбрать модель для анализа из реестра:
- **Встроенные** (без ключа): Gemini 3 Flash/Pro, Gemini 2.5 Pro/Flash/Lite, GPT-5/Mini/Nano/5.2
- **ProxyAPI** (нужен ключ): Claude 4 Sonnet/Opus, GPT-5/5-mini
- **OpenRouter** (нужен ключ): любая модель из каталога

### UX анализа
- Прогресс-лог в реальном времени (определение границ → расчёт страниц → настроение/темп для каждой сцены)
- Фоновый предвыбор (prefetch) до 3 следующих глав
- Возможность повторного/продолжения анализа
- Карточки сцен с цветовой кодировкой по типу, длительностью и раскрываемым превью текста

## 5. Передача главы в Студию ✅

**Модуль:** Парсер → Студия

### Условия доступности
Кнопка переноса (🎬) отображается когда:
- Глава верхнего уровня: сама глава и **все** её дочерние подглавы имеют статус `done` с непустыми сценами.
- Подглава: аналогичная рекурсивная проверка.

### Механизм передачи
1. Собираются сцены из самой главы и всех её дочерних подглав.
2. Для вложенных элементов заголовок формируется как: `«Название родителя (Часть N)»`, где N — порядковый номер.
3. Данные сохраняются в `sessionStorage` (с автоматической деградацией при переполнении — обрезка текста сцен).
4. Пользователь перенаправляется на страницу Студии.

## 6. Студия: навигатор сцен ✅

**Модуль:** Студия (левая панель)

- Отображение списка сцен загруженной главы.
- Цветовая кодировка по типу сцены (action, dialogue, description…).
- Оценка длительности каждой сцены и главы в целом (по количеству символов).
- Индикатор 🎬 для сцен, у которых уже выполнена раскадровка (сегментация).
- Статистика в хедере: общая длительность главы, количество символов.

## 7. Раскадровка (Storyboard) ✅

**Модуль:** Студия → вкладка «Раскадровка» + Edge Function `segment-scene`

### Автоматическая сегментация
При выборе сцены в навигаторе система автоматически:
1. Проверяет, есть ли уже сегменты в БД для данной сцены.
2. Если нет — запускает AI-анализ, разбивающий текст сцены на типизированные сегменты.

### Типы сегментов
| Тип | Описание |
|-----|----------|
| 📖 **Рассказчик** (narrator) | Повествование от третьего лица |
| 👤 **От первого лица** (first_person) | Повествование от первого лица |
| 💬 **Диалог** (dialogue) | Прямая речь персонажа (с указанием спикера) |
| 🧠 **Мысли** (inner_thought) | Внутренний монолог |
| ✍️ **Эпиграф** (epigraph) | Цитата или эпиграф |
| 🎵 **Лирика** (lyric) | Стихотворный или песенный фрагмент |
| 📝 **Сноска** (footnote) | Примечание |

### Структура данных
Каждый сегмент разбивается на **фразы** (предложения) — атомарные единицы для озвучки:
- `scene_segments` → тип, спикер, порядковый номер
- `segment_phrases` → текст фразы, порядковый номер

### Редактирование
- Каждая фраза редактируемая inline (клик → textarea → Enter для сохранения, Esc для отмены).
- Изменения сохраняются в БД мгновенно.
- Возможность переанализа сцены (кнопка «Переанализ»).

## 8. Профилирование персонажей ✅

**Модуль:** Студия → вкладка «Персонажи» + Edge Function `profile-characters`

### Автоматическое извлечение
Во время сегментации сцены (шаг 7) система автоматически:
1. Извлекает всех спикеров из диалоговых сегментов.
2. Создаёт/обновляет записи в `book_characters` (upsert по имени).
3. Создаёт записи в `character_appearances` (связь персонаж ↔ сцена ↔ сегменты).

### AI-профилирование
По запросу (кнопка «AI-Профайлинг») или автоматически при подборе актёров:
1. Собирает реплики каждого персонажа из сегментов.
2. Собирает фрагменты нарратора для контекста.
3. Отправляет в AI (Gemini 2.5 Flash) для определения:
   - **Пол** (male / female / unknown)
   - **Возрастная группа** (child / teen / young / adult / elder)
   - **Темперамент** (sanguine / choleric / melancholic / phlegmatic / mixed)
   - **Стиль речи** (2–3 предложения)
   - **Психологический портрет** (3–5 предложений)
   - **Алиасы** (все варианты имени, прозвища, обращения)

### Инкрементальное профилирование ✅
При загрузке **новой главы** в студию:
1. Новые персонажи → создаётся полный профайл.
2. Известные персонажи, появляющиеся в новых сценах → профайл обновляется только при существенных новых проявлениях характера.
3. Известные персонажи, не появляющиеся в новых сценах → профайл не изменяется.

### Ручное редактирование
Пользователь может вручную изменить:
- Пол (через popover-меню)
- Возрастную группу (через popover-меню)
- Темперамент (через popover-меню)

## 9. Подбор голосов (Кастинг) ✅

**Модуль:** Студия → вкладка «Персонажи» + кнопка «Подбор Актёров»

### Автоматический подбор
Кнопка **«Подбор Актёров»** в хедере вкладок запускает двухэтапный процесс:

**Этап 1 — Кастинг (назначение голосов):**
- На основе пола и возрастной группы каждого персонажа подбирается оптимальный голос Yandex SpeechKit:
  - Женщины: alena, jane, dasha, julia, lera, marina, masha
  - Мужчины: kirill, alexander, zahar, ermil, anton, filipp, madirus
  - Дети: masha (ж), filipp (м)
  - Пожилые: omazh/jane (ж), zahar (м)
- Избегание дублирования голосов между персонажами одного пола.
- На основе темперамента назначается **роль** (интонационный стиль): neutral, friendly, strict, good, evil, whisper.

**Этап 2 — Инкрементальное профилирование:**
- Автоматически запускается после кастинга для уточнения профилей на основе текущих сцен.

### Настройки голоса (на персонажа)
| Параметр | Описание |
|----------|----------|
| Голос | Выбор из реестра Yandex SpeechKit (15+ голосов) |
| Роль | Интонационный стиль (зависит от доступных ролей голоса) |
| Скорость | 0.1 – 3.0 (по умолчанию 1.0) |
| Громкость | Нормализация для балансировки голосов |
| Высота | Тонкая настройка тембра |

### Предпрослушивание ✅
Кнопка ▶ рядом с настройками голоса — синтезирует короткий отрывок через Edge Function `yandex-tts` и воспроизводит в браузере.

## 10. Таймлайн 🔧

**Модуль:** Студия (нижняя панель)

Текущее состояние — визуальный макет с заглушками:
- Линейка времени с масштабированием (zoom in/out, fit to width).
- 4 дорожки: Диктор 1, Диктор 2, Атмосфера, SFX.
- Моковые клипы для демонстрации расположения.
- Drag-resize панели по вертикали.
- Сворачивание/разворачивание.

**⬜ Запланировано:** подключение реальных аудио-клипов, drag-and-drop, микширование.

## 11. Библиотека книг ✅

**Модуль:** Парсер → Библиотека

- Список всех загруженных книг пользователя.
- Для каждой книги: название, дата загрузки, количество глав и сцен.
- Бейдж статуса: «Analyzed» / «Uploaded».

## Общая схема пайплайна

```
PDF-файл
  │
  ▼
┌─────────────────────────────────┐
│  1. Загрузка + хранение         │  Парсер / Библиотека
│  2. Извлечение TOC              │
│  3. Ручная правка структуры     │  Навигатор
│  4. Семантический анализ        │  AI → границы сцен → метаданные
└─────────────┬───────────────────┘
              │  🎬 Передача главы
              ▼
┌─────────────────────────────────┐
│  5. Раскадровка (сегментация)   │  AI → типизация + фразы
│  6. Профилирование персонажей   │  AI → психопортрет
│  7. Кастинг голосов             │  Автоподбор + ручная настройка
│  8. Предпрослушивание           │  Yandex SpeechKit TTS
└─────────────┬───────────────────┘
              │  ⬜ следующие этапы
              ▼
┌─────────────────────────────────┐
│  9.  Запись голосов (TTS)        │  ⬜
│  10. Атмосфера + SFX            │  ⬜
│  11. Монтаж на таймлайне        │  ⬜
│  12. Сведение + мастеринг       │  ⬜
│  13. Экспорт финального аудио   │  ⬜
└─────────────────────────────────┘
```
